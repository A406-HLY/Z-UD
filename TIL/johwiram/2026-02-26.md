## 오늘 한 일

### 1. 주담대 심사 요구사항 명세서 구체화
- 기능 요구사항(F)과 비기능 요구사항(NF)을 체계적으로 정리
- 전체 업무 흐름을 다음 단계로 구조화
  - 서류 입력
  - 서류 자동 분류 및 핵심정보 추출
  - 서류–전산 자동 대조
  - 담보 분석 (권리관계, 선순위 채권, 방공제 등)
  - LTV·DSR 산출
  - 정책성 상품 검증
  - 4단계 판정 (Hard Stop / 보완 / 한도 산출 / 수기심사)
  - 리포트 생성
- 각 기능에 법령·감독규정 근거를 명시하여 규제 기반 설계 형태로 정리

### 2. 기능 간 역할 재정의
- “대조 엔진”과 “규정 기반 판단 로직”의 역할을 분리해야 함을 인지
  - 정보 비교는 기술 기능
  - 판정은 정책·규정 기반 로직
- 실시간 보완 안내(F-007)와 리포트 내 체크리스트(F-029)의 단계 차이 명확화
- Hard Stop / CDD / EDD는 단순 경고가 아니라 분기 로직임을 재정리

### 3. 금융 시스템 관점 보완 포인트 도출
- 내부 신용평가 시스템 연계 요구 필요성 확인
- Hard Stop override 권한 통제 구조 정의 필요성 인식
- OCR 신뢰도 기반 수기 확인 분기 로직 추가 필요성 도출
- 자동화보다 “통제 설계”가 금융 시스템의 핵심임을 재확인
